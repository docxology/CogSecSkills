# CLAUDE.md — CogSecSkills

Guidance for Claude Code working in this repository. This file specializes the
full contributor contract in [`AGENTS.md`](AGENTS.md) for Claude Code sessions —
read `AGENTS.md` for the authoritative rules, the per-folder `AGENTS.md`
hierarchy, and the skill-conformance contract. Overview is in
[`README.md`](README.md); ideal-state in [`ISA.md`](ISA.md).

## What this is

A library of **100 defensive, multiharness agentic skills** for Cognitive
Security and analytic tradecraft (Structured Analytic Techniques, OSINT
integrity, counterintelligence, critical review). Each skill is authored once as
a harness-neutral spec and rendered with default adapters for Claude Code,
Codex, and Hermes. The deliverable is the skills system, not a single app.

## Environment

- Python 3.10+; dependency and run management is through `uv`.
- `uv sync` installs the core runner; `uv sync --extra dev` adds ruff/mypy/pytest;
  `uv sync --extra figures` adds matplotlib/numpy/seaborn (only for regenerating
  figures via `manuscript-assets --write`).
- Run the CLI as `uv run python -m cogsecskills <cmd>` (or with `PYTHONPATH="src:."`).

## The one rule that matters most: never hand-edit generated files

The repo proves Plan↔Build coherence with strict drift `--check` gates. Editing a
generated file by hand will fail those gates. **Generated files** (regenerate via
the listed command — never edit directly):

| Generated file(s) | Regenerate with |
|---|---|
| `skills/<group>/<slug>/{skill.yaml,SKILL.md,workflow.md,harness/*}` | `definitions --write` (source: `definitions/<group>/<slug>.yaml`) |
| `docs/catalogue.md` | `catalogue --markdown --output docs/catalogue.md` |
| `docs/skill-worked-examples.md` | `examples --write` |
| `docs/evaluation-readiness.md` | `evals --write` |
| `docs/quality-dashboard.md`, `docs/quality-dashboard.html` | `dashboard --write` |
| `docs/release-claim-matrix.md` | `release-metadata --write` |
| `manuscript/S10_skill_catalogue.md`, `manuscript/S11_skill_metadata_matrix.md`, `output/figures/*` | `manuscript-assets --write` |

Everything else under `docs/` and `manuscript/` (except the files above) is
hand-authored. Each generated file declares its source command in a top-of-file
`<!-- Generated by … -->` comment — trust that comment.

## After any change, run the gates

```bash
uv run python -m cogsecskills validate          # 0 errors required
uv run python -m cogsecskills doctor            # validate + quality lint
uv run python -m cogsecskills definitions --check
uv run python -m cogsecskills scenarios --check
uv run python -m cogsecskills examples --check
uv run python -m cogsecskills evals --check
uv run python -m cogsecskills dashboard --check
uv run python -m cogsecskills release-metadata --check
uv run python -m cogsecskills manuscript-assets --check   # needs the figures extra
```

If you changed a source that feeds a generated file, run its `--write` first,
then `--check`. CI (`.github/workflows/ci.yml`) additionally enforces:

```bash
uv run ruff check src/cogsecskills tests
uv run ruff format --check src/cogsecskills tests
uv run mypy
uv run pytest --cov=cogsecskills --cov-report=term-missing   # coverage gate >=90%
```

## Conventions

- All logic lives in `src/cogsecskills/`; `definitions/` and `skills/` are
  declarative data; the CLI is a thin orchestrator. No mocks in tests (real
  `tmp_path` + real YAML).
- Do not copy live counts (skill totals, coverage %, test counts) into evergreen
  prose — `100 skills / 7 groups` is pinned intentionally (see `AGENTS.md` →
  "How to add a skill"); other counts should signpost to generated output.
- **Defensive only.** These skills recognize, assess, and defend against
  cognitive attack. Never author manipulation, influence-op playbooks, or
  offensive how-tos. Enforced by review, inherited from AGEINT.

## Using these as Claude Code `/` skills

The `skills/` tree is not directly discoverable by Claude Code (nested groups +
dotted names). To install them into the `/` skill menu, flatten them into
`.claude/skills/cogsec-<slug>/` — see
[`docs/claude-code-skills.md`](docs/claude-code-skills.md). `.claude/skills/` is a
regeneratable copy of `skills/`, not a source.
