# CogSecSkills Design System

CogSecSkills uses a print-first analytic style: dense enough for a 100-skill
library, restrained enough for a manuscript, and high-contrast enough for PDF,
HTML, and screenshot review.

## Principles

- Prefer evidence surfaces over decoration: every visual should answer a reader
  question about skills, harnesses, scenarios, examples, or claim boundaries.
- Keep the palette broad but quiet. Use group colors for taxonomy meaning, not
  as a full-page theme.
- Use readable type before fitting more text. Generated figures and dashboards
  should favor larger labels, direct annotations, and wrapped tables.
- Make claim boundaries visible near evidence summaries. Local gates prove
  synchronization and fixture readiness, not field effectiveness or live runtime
  certification.

## Tokens

| Token | Value | Use |
|---|---|---|
| Surface | `#FCFCFD` | Figure and page background |
| Panel | `#FFFFFF` | Cards, table containers, source panels |
| Ink | `#1F2430` | Primary text, titles, strong rules |
| Muted | `#6F768A` | Secondary labels and caveats |
| Grid | `#E6E8F0` | Table rules, light separators |
| Axis | `#D7DBE7` | Chart axes and subtle borders |

The manuscript figure generator owns the canonical group palette. Any new
visual surface should reuse those colors or document why a neutral treatment is
clearer.

## Generated Dashboard

`python -m cogsecskills dashboard --write` owns:

- `docs/quality-dashboard.md`
- `docs/quality-dashboard.html`
- `output/data/quality_dashboard.json`

The HTML dashboard is static and dependency-free. It should keep semantic HTML,
responsive table wrappers, a print stylesheet, and a visible claim-boundary
panel. Do not add a framework unless the project gains a real interactive site
surface.

## Figures And Cover

Figures are static PNGs generated by
`python -m cogsecskills manuscript-assets --write`. They should remain
PDF-first:

- large canvas dimensions and at least 240 DPI;
- direct labels where possible;
- no tiny legend-only interpretation;
- no decorative gradients or unrelated imagery;
- cover-page insert text large enough to read after manuscript scaling.

## Tables

For manuscript tables, use narrow numeric/status columns and reserve width for
reader-facing prose columns such as quality capsules. For HTML tables, prefer
horizontal scrolling over cramped text on mobile.
