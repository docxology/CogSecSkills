# Getting Started Checklist

The Getting Started Checklist is a structured analytic technique that disciplines analysts to articulate the core question, identify key drivers, surface explicit assumptions, and inventory prior judgments before any substantive analysis begins. It counteracts anchoring and premature closure by creating a documented baseline against which later reasoning can be compared. The checklist operationalizes the discipline of separating problem framing from problem solving, a foundational step in the Heuer-Pherson SAT family.

## When to use

- at the very start of any analytic task before research or drafting begins
- when the question feels confused or the consumer's actual need is unclear
- when a team needs a shared, explicit problem statement to coordinate work
- when reviewing an inherited or ongoing assessment to check framing drift

## What it produces

- an explicit primary question in one answerable sentence
- a list of key drivers — variables whose state most affects the answer
- a register of assumptions with their confidence levels
- a log of prior judgments that could anchor subsequent reasoning
- a gap list: information needed but not yet available

## Key discipline

- state the question in one sentence before doing anything else — vague questions produce vague answers
- distinguish drivers (variables) from assumptions (things treated as true but unverified)
- log prior judgments explicitly so anchoring bias can be recognized and controlled
- the checklist is a living document — revisit it when new information reframes the problem
